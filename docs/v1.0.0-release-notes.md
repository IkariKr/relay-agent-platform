# Relay `v1.0.0` Release Notes

## What This Release Is

`v1.0.0` is the first public release of the multi-surface Relay project.

This release makes the project publishable as a documented, supportable skill family with:

- a unified agent package for Claude and OpenCode routing
- specialist Claude and OpenCode package surfaces
- transparent auto routing
- user-manageable routing rules
- generated package validation and release documentation

## Default Recommendation

New users should start with:

- `relay-agent`

Use the specialist packages only when you intentionally want Claude-only or OpenCode-only behavior.

## Included In `v1.0.0`

### Unified Agent Package

- public unified entrypoint: `scripts/run_delegate_agent.ps1`
- explicit backend override through `-Backend claude|opencode`
- transparent auto routing through `-Backend auto`

### Transparent Routing

- default routing template in `auto-routing.default.json`
- fixed config precedence
- explainable rule selection and backend fallback behavior

### Rule Management

- structured management through `scripts/manage_auto_routing.ps1`
- natural-language wrapper through `scripts/manage_auto_routing_nl.ps1`
- workspace-level routing customization without source edits

### OpenCode Support

- generated OpenCode package
- OpenCode model intent and provider preference controls
- backend previews through `-WhatIf`

### Release And Support Assets

- package build pipeline
- package validation pipeline
- installation guide
- quickstart guide
- routing guide
- troubleshooting guide
- release checklist

## Installation Notes

Recommended source install flow:

```powershell
.\scripts\build-packages.ps1
.\scripts\install-workspace-skill-links.ps1
```

If you only want generated package artifacts, copy:

- `packages\relay-agent`
- `packages\relay-opencode`

into your Codex skills directory as needed.

## Known Limitations In `v1.0.0`

- rule order is top-to-bottom only; there is no priority field
- newly added rules are appended, not auto-reordered
- the natural-language wrapper is intentionally constrained and works best with labeled fields
- there is no GUI or visual rule editor
- there is no analytics or dynamic-learning routing layer

## Recommended Next Step After Install

Use `-WhatIf` first:

```powershell
.\scripts\run_delegate_agent.ps1 -Prompt "Please review this design doc." -Backend auto -WhatIf
```

Then inspect routing:

```powershell
.\scripts\manage_auto_routing.ps1 -Action list -Workdir .
.\scripts\manage_auto_routing.ps1 -Action explain -Workdir . -Prompt "Please review this design doc."
```

## Upgrade Guidance

For source-repo installs:

```powershell
git pull
.\scripts\build-packages.ps1
.\scripts\validate-packages.ps1
```

## Public V1 Positioning

`v1.0.0` should be understood as:

- stable enough for public use
- focused on transparency and supportability
- intentionally conservative on routing complexity

Future releases can improve routing sophistication, but `v1.0.0` locks in a simple, understandable contract first.
